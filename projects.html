This project has been made as a university course work.
This was a very fun project to do and i learned a lot about Design Patterns, SOLID, threads and architectures.
The project is written in Java and can be broken down to 3 main parts.

  • Algorithems

    Algorithems used to generate a maze - simple single path & Randomized Prim's algorithm.
    Searching algorithems - BreadthFirstSearch, BestFirstSearch, DepthFirstSearch.
    Adapter Design Pattern is being used to adapt a Maze instance to a searchable maze.

  • Client-Server architecture

    Server uses a threadpool to handle multiple clients and Strategy Design Pattern to determine at run-time two kinds of client requests - Maze generation and Solution to the maze. A byte array representing the maze is being compressed between communications by convering binary data(maze is represented as 0's and 1's) to 8-bit integers.
    A singleton configuration file determines which kind of maze generation and solution algorithem to apply.

  • GUI and MVVM Architecture

    The first two parts can be created as a JAR and used here, but for simplicity and comfort i uploaded to GitHub all of the packages.
    Creation of GUI using JavaFX and SceneBuilder. MVVM architecture is used with the implementation of observer design pattern.
    Includes music, zoom, diagonal movement of the character,visual solution path and saving\loading mazes.
